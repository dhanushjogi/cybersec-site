# Security Policy

## Reporting a Vulnerability

We take security vulnerabilities very seriously. If you discover a security vulnerability in cybersec-site, please email us at dhanushjogi33.dj@gmail.com with the following information:

1. Description of the vulnerability
2. Steps to reproduce
3. Potential impact
4. Suggested fix (if any)

## Do not disclose the vulnerability publicly until we have had a chance to fix it.

## Supported Versions

We provide security updates for the following versions:

| Version | Supported |
|---------|----------|
| 1.0.x   | Yes      |
| < 1.0   | No       |

## Security Measures

Our project implements the following security measures:

- CodeQL static analysis enabled
- Dependabot automated dependency updates
- Secret scanning enabled
- Input validation and sanitization
- Parameterized database queries using Mongoose
- Password hashing with bcrypt
- JWT-based authentication

## Security Response Time

We aim to respond to security vulnerability reports within 48 hours and provide a fix within 7 days when possible.
